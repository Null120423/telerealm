Protected API and public `/link/{botToken}/{chatID}` integration

TeleRealm supports both bearer-token protected endpoints and a path-based public API for scoped CRUD file records.

Tip

Public path parameters expose your bot token in the URL.
Use this mode only for trusted or internal integrations.
Prefer protected endpoints for production-grade security.

Protected API

POST /send
Authorization: Bearer <botToken>
FormData: chat_id, document

GET /url?file_id=<fileID>
GET /info?file_id=<fileID>
GET /verify?chat_id=<chatID>

Public CRUD API

POST   /link/{botToken}/{chatID}
GET    /link/{botToken}/{chatID}
GET    /link/{botToken}/{chatID}/{recordID}
PATCH  /link/{botToken}/{chatID}/{recordID}
DELETE /link/{botToken}/{chatID}/{recordID}

Upload Example

curl -X POST \
  -F "document=@/tmp/photo.png" \
  http://localhost:7777/link/123456:ABCDEF/123456789

Update Example

curl -X PATCH \
  -H "Content-Type: application/json" \
  -d '{"original_name":"hero-banner.png"}' \
  http://localhost:7777/link/123456:ABCDEF/123456789/<recordID>

Delete Behavior

Delete removes TeleRealm's in-memory record only. It does not remove the underlying Telegram file from storage.

Response Shape

{
  "success": true,
  "message": "Public upload completed successfully!",
  "data": {
    "record_id": "uuid",
    "file_id": "telegram-file-id",
    "chat_id": "123456789",
    "url": "https://api.telegram.org/file/...",
    "secure_url": "http://localhost:7777/drive/...",
    "bytes": 12345,
    "format": "png",
    "original_name": "photo.png"
  }
}
